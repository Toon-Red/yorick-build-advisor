## KEY DESIGN PATTERNS FOR OUR APP

### What Mobafire Does Well
1. **Multiple options with labels** — 10 rune pages, 3 skill orders, 15+ item builds, all named and described
2. **Tabbed navigation** — Click tabs 1-10 for runes, 1-2-3 for skill order
3. **Threat/synergy tier list** — Visual champion icons with 5 threat levels
4. **Matchup-specific item builds** — Not just generic builds, but "vs Jax do THIS"
5. **Creator voice** — Kampsycho's personality comes through (jokes, slang, strong opinions)
6. **Deep mechanics** — Ghoul AI, wall interactions, heal math all explained
7. **Combo naming** — "Big Black Cleaver", "Titanic Breaker", "Become a Believer" — memorable

### What Mobafire Does Poorly (Our Opportunity)
1. **Information overload** — User must scroll through 15+ builds, 10 rune pages, and figure out which applies to THEIR matchup
2. **No filtering** — If I'm vs Jax, I still see all 15 builds. Creator wrote "Iceborn Cleaver vs Jax" but it's buried
3. **No decision tree** — Creator has the knowledge ("vs immobile use Tab 1 skill order") but user must read and decide
4. **Matchup notes are sparse** — Most threats just say the champion name with no advice
5. **No per-matchup skill order** — 3 generic tabs, user must know "Jax = immobile = Tab 1"
6. **Static page** — Can't interact, can't auto-detect champ select, can't import to client
7. **No structured reasoning** — Creator writes paragraphs but can't attach reasoning to specific decisions

### Our App's Value Proposition
**Same creator knowledge, but FILTERED through a matchup selector.**

Instead of: "Here are 10 rune pages, figure out which one"
We show: "You're vs Jax → Here's Grasp (primary), Comet (alternative), with reasoning"

Instead of: "Here are 15 item builds somewhere on the page"
We show: "vs Jax → Iceborn Cleaver path, because [creator's reasoning]"

Instead of: "3 skill order tabs, read the descriptions"
We show: "vs Jax → Tab 1 (W-stack), because he's immobile with one dash"

## DATA MODEL FOR GUIDE CREATOR SUPPORT

### What Creators Need to Express

From analyzing this Mobafire guide, creators need to be able to define:

1. **Rune pages** (named, with full selections)
   - 10 pages here, each with primary/secondary/shards
   - Each page has a descriptive name + "when to use" text

2. **Skill orders** (multiple, named, with conditions)
   - 3 here: W-stack (immobile), Standard Q-max (default), E-max (ranged/comet/aery)
   - Each has: level-by-level order [1-18] + description + condition ("vs immobile", "vs ranged + comet/aery")

3. **Item builds** (named, with reasoning)
   - Starters, first back, core, situational, boots
   - Named combos with synergy explanations
   - Matchup-specific overrides ("vs Jax: Iceborn first")

4. **Matchup table** (per-enemy)
   - Threat level (Extreme/Major/Even/Minor/Tiny)
   - Recommended keystones (ordered by priority)
   - Recommended skill order tab
   - Recommended item build path
   - Summoner spells
   - Starter items
   - Freeform advice text (creator's reasoning, tips, tricks)
   - Shard overrides (MS vs HP etc.)

5. **Synergies** (ally champions that pair well)
   - Tier + reasoning

6. **Guide text** (ability breakdowns, strategy, mechanics)
   - Per-ability explanations
   - General strategy
   - Creator personality/voice

### Proposed Schema for Guide Files

A guide file should let creators define all the above, with matchups LINKING to the named options:

```
Guide:
  champion: "Yorick"
  author: "Kampsycho"

  rune_pages:
    - id: "grasp_default"
      name: "Default Tank/Grasp Build"
      primary: [Grasp, Demolish, BonePlating, Revitalize]
      secondary: [Alacrity, PresenceOfMind]
      shards: [AS, HP, Tenacity]

  skill_orders:
    - id: "w_stack"
      name: "W-Stack vs Immobile"
      order: [Q,E,W,Q,W,R,W,Q,Q,_,R,E,E,E,E,R,W,W]
      description: "3 points W by level 7, then Q max, E max, W last"

  item_builds:
    - id: "iceborn_cleaver"
      name: "Iceborn Cleaver"
      starter: "cloth_refillable"
      core: [IcebornGauntlet, BlackCleaver, SunderedSky]
      situational: [SpiritVisage, Shojin, Steraks]

  matchups:
    Jax:
      difficulty: "EXTREME"
      rune_pages: ["grasp_default", "comet_poke", "aery_ranged"]
      skill_order: "w_stack"  # immobile, one dash
      item_build: "iceborn_cleaver"
      summoners: "Exhaust viable (Ghost/Ignite default)"
      starter: "cloth_refillable"
      shard_override: {row2: "MS"}
      advice: "Wall after he dashes, 3W by lvl 5. Bait jump, wall him, trade, back off."
```

This lets:
- **Creators** define their full knowledge in a structured way (like Mobafire but organized)
- **The engine** filter to just what's relevant when user selects "vs Jax"
- **The UI** show ranked options with creator reasoning attached
- **LCU integration** import exact rune/item IDs

### Guide Editing UX (Future)

Creators should be able to:
- Add/edit rune pages with a visual builder (like Mobafire's)
